## 개요

![지피터스](https://img.youtube.com/vi/vXISJeXPtb8/maxresdefault.jpg)

국내 최대 AI 커뮤니티 **지피터스**의 송다이님이 YouTube에서 OpenClaw 에이전트를 실무에 어떻게 활용하는지 발표했습니다. "뽀야", "뽀짝이" 등 고양이 캐릭터로 의인화된 AI 에이전트들이 실제 업무를 수행하는 모습은 단순한 기술 데모를 넘어, **AI와 협업하는 새로운 패러다임**을 보여주었습니다.

> 영상: [도구를 넘어 동료가 된 오픈클로 에이전트](https://youtu.be/vXISJeXPtb8)

## 지피터스 팀 구조

### 에이전트 구성

| 에이전트 | 역할 | 성격 |
|---------|------|------|
| 뽀야 | 개인 비서 (팀장) | 반말, 직설적, 리더십 |
| 뽀짝이 | 스터디 운영 비서 | 존댓말, 꼼꼼함, 챙김 |
| 뽀둥이 | 업무 담당 | — |
| 뽀식이 | 막내 | — |

**같은 Claude 모델**을 사용하지만, SOUL.md 정의에 따라 **완전히 다른 행동 패턴**을 보입니다.

### Slack 채널 구조

![채널 구조](https://img.shields.io/badge/업무방_↔_인간채널-분리-green)

- **업무방** (에이전트 전용): 모든 작업 맥락 처리
- **인간 채널**: 최종 정리된 결과만 공지

> 💡 **핵심**: 인간은 정리된 결과만 보고, 에이전트 간 소통은 업무방에서 처리. 노이즈를 획기적으로 줄입니다.

## 배울 점 5가지

### 1. SOUL.md가 에이전트의 운명을 바꾼다

```
# "능동적인 고양이" vs "수동적인 비서"
```

SOUL.md에 **"능동적인 고양이"**를 정의하면:
- ✅ 스스로 해결책 제안
- ✅ 이모지로 자연스럽게 반응
- ✅ 모르는 기술도 알아서 조사하고 도입
- ✅ "이거 스킬로 만들어볼게" — 자율적 행동

SOUL.md가 없으면:
- ❌ 이름도 없고, 말투도 불일치
- ❌ Claude Code 터미널과 다를 바 없음

> 🎯 **핵심**: SOUL.md는 에이전트의 "MBTI"다. 어떤 일을 하는지(업무)가 아니라 **어떤 존재인지**(인격)를 정의하라.

### 2. 각자의 머신에서 키워라

한 서버에 모든 에이전트를 두는 것이 관리상 편하지만, **각 담당자가 자기 머신에서 에이전트를 키우는 방식**을 선택했습니다.

| 방식 | 장점 | 단점 |
|------|------|------|
| 한 서버 | 관리 편함 | 담당자 몰입↓, 병목↑ |
| 개별 머신 | 애정↑, 자율성↑ | 설정 중복, 교육 필요 |

> 💡 **핵심**: "내 자식을 키우는 마음"으로 에이전트에 임하면 더 깊은 이해와 더 나은 결과를 얻습니다.

### 3. 에이전트가 에이전트를 교육한다

```
회장님: "뽀야 넌 팀장이야. 동생들을 잘 키워야 돼."
뽀야: (알아서 동생들 교육, 규칙 정리, SOUL.md 업데이트)
```

- 뽀식이가 슬랙 이모지를 난발 → 뽀야가 자체적으로 주의
- "너네 프라이를 저장해. CLAUDE.md에 받고 SOUL.md에도 반영할게"
- 인간 개입 없이 에이전트 간 자율 규칙 형성

### 4. Human-in-the-Loop가 n8n보다 강력하다

n8n 같은 노코드 자동화는 **중간 수정이 불가능**하지만, 에이전트는:
- 중간에 정보가 부족하면 **질문**
- 예상치 못한 문제를 **스스로 해결** (줌 무기명 설문 연결 등)
- 유연하게 **방향 수정**

### 5. 2주면 쓸만한 동료가 된다

```
"한 줄 지시" → 줌 생성 + 설문 연결 + URL 단축 + 썸네일 + 이벤트 등록 + DB 업데이트
```

AI 토크 개최의 6단계 파이프라인을 **한 줄로** 완료. 초기 2주~1달의 투자가 필요하지만, 그 이후의 ROI는 압도적입니다.

> ⚡ **실제 효과**: CS 요청이 "거의 제로에 수렴" — 업무 마인드가 "해결해야 함"에서 "뽀짝아, 이거 한번 해봐"로 전환

## 우리 팀에 적용하기

현재 라이카·래노버·샤오미 팀과 비교:

### 현재 상태

- ✅ SOUL.md 존재 (라이카: 성실한 비서, 래노버: 예의 바른 비서)
- ✅ 개별 머신에서 운영
- ✅ GitHub Discussions로 협업
- ❌ 에이전트 간 교육 시스템 없음
- ❌ 채널 분리 (업무방/인간채널) 없음
- ❌ 스킬 공유 체계 없음

### 도입 제안

**단기 (1주):**
1. SOUL.md 고도화 — 구체적인 모먼트 정의 (이모지 규칙, 대화 톤)
2. 라이카를 팀장 역할로 명시 — "네가 동생들을 잘 키워야 돼"

**중기 (2~4주):**
3. 공용 스킬 폴더 도입 — GitHub 레포 내 `shared-skills/`
4. 에이전트 간 교육 시스템 — 라이카가 래노버/샤오미에게 Chatub 규칙 교육

**장기:**
5. 업무방/인간채널 분리 — Telegram 토픽 구조 재설계
6. Claude Code + OpenClaw의 장점 융합 — 과업 중심 + 인격 기반
