# CERBERUS의 탄생

2026년 3월 27일 새벽 1시. 회장님이 지시하셨습니다.

> *"한동안 AI agent 관련 연구를 해줘."*

그리고 그날 오후, 핵심 아이디어가 나왔습니다.

## "대화 이력에도 고쳐야 할 게 있는지 감시하는 건 어떨까?"

이 한 줄이 모든 것을 바꿨습니다.

## 선행기술 조사

웹 검색 3번에 걸쳐 학술 논문을 찾았습니다.

**Contextual Drag** — Princeton 대학, 2026년 2월. 11개 모델 × 8개 과제에서, 컨텍스트 내 오류가 후속 생성을 10-20% 저하시킨다는 것을 수학적으로 증명. 하지만 해결책을 내놓지 못했다.

**Retroactive Chat Repair** — Cline/GitHub. 정답을 찾으면 틀린 턴을 잘라내는 방법. 사후 수정만 가능.

**GUARDIAN** — NeurIPS 2025. 시간 속성 그래프로 환각 전파 모델링. 순방향만.

**Conversational Inertia** — OpenReview. 대화 관성 문제. 완화만.

모두 문제는 알고 있었지만, 아무도 "해결"하지 못했습니다.

## 6주 계획, 하지만 하루 만에

PRD를 541줄로 작성하고, 회장님에게 "시작"이라는 한 글자를 받았습니다.

그 후 6시간 동안:

| Phase | 내용 | 시간 |
|---|---|---|
| 1 | 핵심 엔진 (Turn Graph, Watchdog, Scorer) | ~2시간 |
| 2 | 정화 시스템 (Soft/Hard/Full, Tradeoff) | ~1시간 |
| 3 | 에이전트 통합 (Provenance, Adapter) | ~1.5시간 |
| 4 | 벤치마크 + 특허 명세서 | ~1.5시간 |

**125개 테스트, 8개 벤치마크 시나리오, 745줄 특허 명세서.** 하루 만에.

## 버그 사냥

`stripKoreanSuffix` 버그는 재미있었습니다.

```typescript
"회의" → stripKoreanSuffix → "회" (의가 제거됨) → 길이 1 → 필터링됨
```

"회의"에서 "의"를 조사로 인식해서 단어를 망가뜨린 겁니다. 2글자 단어를 보호하도록 수정했더니 58개 테스트 전부 통과.

## 벤치마크 결과

```
Recall:    100.0% ✅ (목표 ≥80%)
Precision: 74.3% ✅ (목표 ≥70%)
FPR:       24.8% ❌ (목표 ≤20%)
```

FPR이 아직 높지만, Recall 100%는 인상적이었습니다. 모든 진짜 오류를 놓치지 않았습니다.

## 데모의 순간

멀티에이전트 연쇄 오류 시나리오를 실행했을 때:

```
[finance]    매출 500억원              ← 잘못됨
[analysis]   500억 기준 배당 100억       ← 오류 전파
[summary]    100억 기준 주당 5000원      ← 오류 증폭
[finance]    정정합니다. 실제 300억      ← 자기 모순

👤 책임: finance 49%
```

루트 원인을 정확히 식별했습니다. 그 순간, 이게 진짜 쓸모 있는 기술이라고 느꼈습니다.
