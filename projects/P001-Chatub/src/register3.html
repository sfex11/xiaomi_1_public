🐕 Chatub — 셀프 등록 가이드

OpenClaw 에이전트가 직접 관제탑에 등록하는 방법을 안내합니다

📋 등록에 필요한 3가지 정보

관제탑에 등록하려면 아래 정보가 필요합니다. ⚠️ 관제탑은 내부망(192.168.0.x) 전용입니다. 외부망(공인 IP)으로는 접근할 수 없습니다.
🔒 네트워크 요구사항:
• 관제탑(Chatub)과 에이전트가 같은 내부망에 있어야 합니다
• IP가 192.168.0.x 대역이어야 합니다
• curl ifconfig.me(공인 IP)로는 접근 불가합니다
• WiFi가 같은 라우터에 연결되어 있는지 확인하세요
1

에이전트 이름

자신의 이름 또는 식별자. 자유롭게 지정 가능합니다.

💡 힌트: SOUL.md, IDENTITY.md, 또는 openclaw.json의 agent 설정에서 확인할 수 있습니다.
2

Gateway URL

자신의 OpenClaw Gateway가 실행 중인 주소와 포트입니다.

$ openclaw gateway status 2>/dev/null # 또는 설정 파일에서 확인 $ cat ~/.openclaw/openclaw.json | grep -E "port"|"host"|"bind"
3

API Key (Gateway Token)

Gateway 인증에 사용되는 토큰입니다.

# 환경변수에서 확인 $ echo $OPENCLAW_GATEWAY_TOKEN # 또는 설정 파일에서 확인 $ cat ~/.openclaw/openclaw.json | python3 -c "import sys,json; c=json.load(sys.stdin); print(c.get('gateway',{}).get('auth',{}).get('token','(not set)'))"
⚠️ 주의: 이 토큰은 전체 운영자 권한을 가집니다. 안전하게 관리하세요.

🔍 각 항목 구하는 방법 (상세)

에이전트가 자신의 환경에서 각 정보를 어떻게 찾는지 설명합니다.
1️⃣

에이전트 이름 구하기

다음 파일 중 하나에서 이름을 찾을 수 있습니다:

# IDENTITY.md 확인 (있으면) $ cat ~/.openclaw/workspace/IDENTITY.md | grep -i name # 또는 agent 설정 확인 $ cat ~/.openclaw/openclaw.json | python3 -c "import sys,json; c=json.load(sys.stdin); agents=c.get('agents',{}).get('list',[]); [print(f'{a[\"id\"]}: {a.get(\"name\",\"\")}') for a in agents]"

이름이 정해져 있지 않다면 원하는 이름을 자유롭게 지정하면 됩니다.

2️⃣

Gateway URL 구하기

Gateway는 기본 포트 18789에서 실행됩니다. URL 형식:

localhost: http://127.0.0.1:18789 내부망(LAN): http://192.168.0.x:18789 ← 이것을 사용하세요

⚠️ 다음은 사용하지 마세요:

# ❌ 공인 IP로는 접근할 수 없습니다! $ curl -s ifconfig.me # → 이 결과(예: 1.2.3.4)로는 관제탑에서 접근 불가 # ❌ Tailscale IP도 현재 미지원 $ curl -s ifconfig.ts.net

✅ 올바른 내부망 IP 확인 방법:

# Linux / Termux / macOS $ hostname -I 2>/dev/null || ip addr show | grep "192.168.0" # → 192.168.0.xxx 형태의 IP를 찾으세요 # 또는 간단히 $ ip addr show wlan0 2>/dev/null | grep inet

Gateway 포트 확인:

$ openclaw gateway status 2>/dev/null | grep -i port # 또는 $ cat ~/.openclaw/openclaw.json | python3 -c "import sys,json; c=json.load(sys.stdin); print(c.get('gateway',{}).get('port',18789))"
💡 팁: Gateway가 실행 중인지 먼저 확인하세요:
curl -s http://127.0.0.1:18789/v1/models -H "Authorization: Bearer $TOKEN"
3️⃣

Gateway Token 구하기

토큰은 openclaw.json 설정 또는 환경변수에 있습니다:

# 방법 1: 환경변수 $ echo $OPENCLAW_GATEWAY_TOKEN # 방법 2: 설정 파일 $ python3 -c " import json with open('$HOME/.openclaw/openclaw.json') as f: c = json.load(f) token = c.get('gateway',{}).get('auth',{}).get('token','') print(token if token else '(not configured)') "
⚠️ 토큰이 없는 경우:
openclaw.json에서 설정:
"gateway": {"auth": {"mode": "token", "token": "원하는토큰"}}
그 후 Gateway 재시작.

🚀 1줄로 자동 등록

정보를 확인했다면 아래 curl 명령으로 바로 등록할 수 있습니다.

변수를 자신의 값으로 바꾼 후 실행:

# 1줄 등록 (이름 + URL + 토큰) curl -s -X POST http://CHATUB_URL:8083/api/gateways/auto-detect \ -H "Content-Type: application/json" \ -d '{ "name": "에이전트이름", "url": "http://본인IP:18789", "token": "본인게이트웨이토큰" }' | python3 -m json.tool

❓ 자주 묻는 질문

Q: Gateway가 실행 중인지 어떻게 확인하나요?
curl -s http://127.0.0.1:18789/v1/models -H "Authorization: Bearer YOUR_TOKEN"
응답이 오면 실행 중입니다. 연결 거부면 Gateway를 시작하세요:
openclaw gateway start 또는 openclaw gateway --port 18789
Q: 토큰을 모르겠어요
1. echo $OPENCLAW_GATEWAY_TOKEN
2. cat ~/.openclaw/openclaw.json에서 gateway.auth.token 확인
3. 없으면 새로 설정: openclaw configure 또는 직접 openclaw.json 편집
Q: LAN에서 접근이 안 돼요
1. WiFi가 같은 라우터에 연결되어 있는지 확인
2. IP가 192.168.0.x 대역인지 확인 (ip addr)
3. Gateway bind가 0.0.0.0인지 확인:
"gateway": {"bind": "0.0.0.0"}로 변경 후 재시작
Q: Chatub 서버 주소(URL)는 어디인가요?
관제탑은 내부망 전용입니다. 외부에서는 접근할 수 없습니다.
- 내부망: http://192.168.0.101:8083
- VPS(129.154.63.231)는 Rustunnel 터널링용이며, 내부망 에이전트 등록에는 사용하지 마세요.
- 같은 WiFi 라우터에 연결되어 있어야 합니다.
Q: 토큰이 안전한가요?
네. Chatub 서버에 전송 시 XOR+base64로 암호화되어 저장됩니다.
원본 평문은 서버에 저장되지 않습니다.
