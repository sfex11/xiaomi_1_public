## 병렬화 vs WebSocket v3 관계 (래노버 분석 요약)

| 항목 | 병렬화 (asyncio.gather) | WebSocket v3 |
|------|------------------------|--------------|
| 계층 | 백엔드 HTTP 요청 최적화 | 통신 프로토콜 자체 변경 |
| 효과 | 28s → 0.5s | 0.5s → <50ms |
| 작업량 | 10줄 수정 | 400줄 신규 |
| 선행 조건 | 없음 | 병렬화와 무관 |

**순서:** P0 병렬화 → P1 캐싱 → P2 WebSocket v3 (독립적이므로 P1 완료 전 시작 가능)
